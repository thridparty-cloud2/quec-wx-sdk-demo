# quec-wx-sdk-demo

微信小程序的 Quectel SDK Demo 工程。

## 环境要求

- 微信开发者工具
- 已安装 Node.js
- 推荐使用微信开发者工具自带的 npm 构建能力

## 工程导入

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 目录选择本仓库根目录 `quec-wx-sdk-demo`。
4. AppID 使用 `project.config.json` 中配置的 AppID，或替换为你自己的小程序 AppID。
5. 导入后检查 `app.json`、`env.js` 和插件配置是否符合当前环境。

## 依赖安装

项目依赖已写入 `package.json`，首次导入或依赖变更后需要重新安装并构建 npm。

### 方式一：微信开发者工具

1. 在开发者工具中打开项目。
2. 点击“工具” -> “构建 npm”。
3. 构建完成后确认 `miniprogram_npm/` 已生成或更新。

### 方式二：命令行安装

在项目根目录执行：

```bash
npm install
```

随后回到微信开发者工具执行“构建 npm”。

## 配置说明

`env.js` 保存了当前 Demo 的基础业务配置，例如标题、Logo、用户域、布局和协议版本等。接入到不同环境时，优先按实际业务修改该文件中的参数。

## 常见问题

- 如果页面报找不到依赖，先确认是否已执行“构建 npm”。
- 如果自定义组件或分包页面不生效，检查 `app.json` 中对应路径是否与目录结构一致。
- 如果使用插件失败，确认小程序后台已添加对应插件权限，并检查 `app.json` 的 `plugins` 配置。
